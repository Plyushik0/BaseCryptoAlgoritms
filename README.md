# Проект по Криптографии на Go

Этот проект представляет собой набор реализаций различных криптографических алгоритмов и протоколов, написанных на языке Go. Он был создан в образовательных и исследовательских целях для изучения принципов работы криптографических систем.

## Реализованные компоненты:

В рамках проекта реализованы следующие модули:

1.  **Схема разделения секрета Шамира (`SHAMIR/`)**
    *   Реализует криптографическую схему, позволяющую разделить секрет на несколько частей (долей) таким образом, что для восстановления секрета необходимо собрать определенное пороговое количество этих долей. Любое меньшее количество долей не дает информации о секрете.

2.  **Схема групповой подписи (`GROUPEsign/`) (есть проблемы с сериализацией и передачей данных, невенрный результат)**
    *   Реализует протокол, позволяющий любому члену группы подписывать сообщения от имени всей группы. При этом подпись анонимна (не раскрывает, какой именно член группы ее создал), но существует менеджер группы (лидер), который в случае необходимости может установить личность подписавшего. Включает также взаимодействие со службой временных меток для подтверждения времени создания подписи.

3.  **Протокол обмена ключами SPEKE и OTP (`EXPKEYS/`)**
    *   Реализует протокол SPEKE (Simple Password Exponential Key Exchange) для выработки общего сессионного ключа на основе разделяемого пароля. Протокол усилен использованием одноразовых паролей (OTP) по схеме Лампорта для аутентификации сторон перед выполнением SPEKE.

4.  **Протокол распределения ключей Блома (`BLOMH/`)**
    *   Реализует схему симметричного распределения ключей, предложенную Бломом. Доверенный центр генерирует и распространяет секретную информацию таким образом, что любые два пользователя в системе могут самостоятельно вычислить общий секретный ключ для защищенной коммуникации, при этом этот ключ неизвестен другим пользователям.

5.  **Протокол защищенной передачи ключа (`KEYGIVE/`)**
    *   Реализует протокол для безопасной передачи сессионного ключа от одной стороны (отправителя) к другой (получателю). Для защиты сессионного ключа используется асимметричное шифрование (например, RSA), а для обеспечения целостности и аутентичности передаваемого сообщения — цифровая подпись.

6.  **Протокол идентификации Фейга-Фиата-Шамира (`FSID/`)**
    *   Реализует протокол доказательства с нулевым разглашением, позволяющий одной стороне (Prover) доказать другой стороне (Verifier), что она обладает некоторым секретным знанием, не раскрывая сам секрет. Используется схема, основанная на сложности извлечения квадратного корня по модулю большого составного числа.

7.  **Протокол Диффи-Хеллмана (`DF/`)**
    *   Реализует классический алгоритм Диффи-Хеллмана для установки общего секретного ключа между двумя сторонами по незащищенному каналу связи. Безопасность протокола основана на сложности вычисления дискретного логарифма в конечном поле.

8.  **Цифровая подпись Эль-Гамаля с временной меткой (`ELGdigitalsign/`)**
    *   Реализует схему цифровой подписи Эль-Гамаля, основанную на сложности вычисления дискретных логарифмов. Включает взаимодействие с сервером временных меток для получения криптографического подтверждения времени создания подписи.

9.  **Цифровая подпись Фиата-Шамира с временной меткой (`FIATdigitalsign/`) (проблемы с реализацией)**
    *   Реализует схему цифровой подписи, основанную на протоколе идентификации Фиата-Шамира (доказательство с нулевым разглашением). Также включает получение временной метки для подписи.

10. **Протокол аутентификации на основе RSA (`AUTHRSA/`)**
    *   Реализует протокол взаимной аутентификации двух сторон с использованием криптосистемы RSA. Стороны обмениваются зашифрованными сообщениями, содержащими случайные числа и идентификаторы, для подтверждения своей подлинности.

11. **Вспомогательные библиотеки:**
    *   `mylib/mylib.go`: Содержит реализации различных математических функций, используемых в криптографических алгоритмах (расширенный алгоритм Евклида, быстрое возведение в степень по модулю, тесты простоты, КТО и т.д.).
    *   `hashf/sha.go`: Пользовательская реализация алгоритмов хеширования SHA-256 и SHA-512.
    *   `cipher` : Функции для симметричного и асимметричного шифрования (например, RSA, Эль-Гамаль).

## Структура проекта:

Каждый основной криптографический протокол или схема вынесен в отдельную директорию. Общие вспомогательные функции находятся в директориях `mylib` и `hashf`.

## Цели проекта:

*   Изучение и понимание фундаментальных криптографических концепций.

*   Демонстрация взаимодействия различных криптографических компонентов.
